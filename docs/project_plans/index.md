# Project Plans

Multi-file, multi-session project trackers. A **project plan** is a doc-truth roadmap for a body of
work larger than a single note — it holds the originating brief, the traced code map, the sectioned
plan, live status, serialized notes/mini-decisions, an append-only action log, and a cold-start
orientation.

New here? Read `PROJECT_PLAN_GUIDE.md` (same folder) before creating or managing a project. Each
project is a set of files sharing a `NNN_` prefix (zero-padded serial, incrementing per project) and
a topic slug.

# Documents

* [Project Plan Guide](PROJECT_PLAN_GUIDE.md) — the self-contained lifecycle guide: setup, upkeep, and closing rules for this folder.
